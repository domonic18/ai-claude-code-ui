# 记忆系统功能需求文档

## 一、需求概述

为 Claude Code UI 系统添加记忆管理功能，实现长期记忆自动化更新，帮助 AI 保留用户偏好、工作习惯和关键决策，提升对话的连贯性和个性化体验。

## 二、核心功能

### 2.1 长期记忆 (MEMORY.md)

**目的**：存储跨对话可复用的用户偏好和工作习惯

**核心功能**：
- 读取/写入/追加/删除长期记忆
- 获取统计信息（字数、章节数）

**文件结构**：
- 路径：`/workspace/memory/MEMORY.md`
- 格式：Markdown
- 标准章节：
  - 审查风格
  - 常见问题答复规则
  - 用户偏好
  - 工作模式偏好

---

### 2.2 记忆上下文加载

**目的**：按优先级构建 AI 对话上下文

**优先级顺序**（从高到低）：
1. 当前用户命令
2. 长期记忆 (MEMORY.md)
3. 项目上下文
4. Skills

**核心逻辑**：
- 按优先级加载内容
- Token 预算管理（超限时从低优先级开始截断）
- 显示记忆来源和估算 Token 使用量

---

### 2.3 智能记忆更新

**目的**：使用 LLM 分析对话内容并智能更新长期记忆

**核心逻辑**：

1. **价值判断**：分析对话是否包含长期可复用信息
   - 有价值：跨对话偏好、长期工作习惯、特定背景信息
   - 无价值：临时工作记录、日常闲聊、短期任务

2. **质量检查**：检查长期记忆是否需要重组
   - 格式混乱、内容冗余、结构问题 → 触发重组

3. **更新模式**：
   - `incremental`（增量）：添加新信息或修改特定章节
   - `reorganize`（重组）：返回完整的重组后内容

**触发时机**：
- 会话结束后延迟一段时间（默认 1 小时）自动触发
- 读取容器内当天的所有对话记录进行分析

**LLM 集成**：
- 支持智谱 API（优先）

---

## 五、智能更新规则

### 5.1 价值判断标准

**有价值 (hasValue=true)**：
- 跨对话可复用的偏好或规则
- 长期的工作习惯或模式
- 需要记住的特定背景信息

**无价值 (hasValue=false)**：
- 临时工作记录、bug 修复
- 日常闲聊、心情日记
- 一两句话且无具体含义
- 纯符号、重复字符

### 5.2 长期记忆质量检查

**检查项**：
- 格式：标题层级、段落重复
- 内容：重复表述、矛盾信息、过时内容
- 结构：缺少核心章节、章节顺序不当

---

## 七、SDK 集成

**文件**：`backend/services/container/claude/ScriptBuilder.js`

**集成点**：
- 读取容器内的记忆文件
- 加载并构建记忆上下文
- 记忆上下文自动注入到每次 AI 查询中


## 九、配置要求

### 环境变量

| 变量 | 说明 | 必需 |
|------|------|------|
| ZHIPU_BASE_URL | 智谱 API 基础 URL | 是 |
| ZHIPU_API_KEY | 智谱 API 密钥 | 是 |
| ZHIPU_MODEL | 智谱模型名称 | 是 |
| AUTO_MEMORY_UPDATE_DELAY_MS | 记忆更新延迟时间（毫秒） | 否，默认 3600000 |

## 十、注意事项

1. **容器环境**：记忆文件存储在容器内 `/workspace/memory/`
2. **文件操作**：通过容器文件操作 API 读写
3. **Markdown 格式**：便于 AI 理解
4. **自动更新**：会话结束后延迟一段时间自动分析对话并更新长期记忆

# Batch Patent Generator 稳定性测试方案

## 1. 测试目标

验证 `batch-patent-generator` agent 在长时间运行（≥6小时）下的稳定性，确保：
- **功能过程异常中断率 < 30%**
- **文档生成成功率 ≥ 70%**
- **断点续传机制正常工作**
- **系统资源使用合理**

## 2. 工作流特性分析

### 2.1 工作流结构

```
阶段1: 生成3种完整论文（工程、科技、经济）
  ↓ 约 30-60 分钟
阶段2: 创意挖掘（从3篇论文中提取3-5个创意）
  ↓ 约 10-20 分钟
阶段3: 批量专利生成（为每个创意生成专利）
  ↓ 约 60-120 分钟
阶段4: 批量方案生成（每个创意 × 4种方案）
  ↓ 约 60-120 分钟
总计: 约 2-4 小时/次
```

### 2.2 文档生成数量

假设挖掘出 **5个创意**：
- **阶段1**: 3篇论文（工程、科技、经济）
- **阶段2**: 1个创意清单
- **阶段3**: 5个专利
- **阶段4**: 5个创意 × 4种方案 = 20篇方案

**总计**: 3 + 1 + 5 + 20 = **29篇文档**

### 2.3 涉及的 Skills

| 阶段 | Skill | 数量 |
|-----|-------|------|
| 阶段1 | paper-engineering-paper | 1 |
| 阶段1 | paper-science-paper | 1 |
| 阶段1 | paper-economy-paper | 1 |
| 阶段3 | patent-patent-writing | N (3-5) |
| 阶段4 | patent-product-plan | N |
| 阶段4 | technical-tech-solution | N |
| 阶段4 | patent-operation-plan | N |
| 阶段4 | patent-ip-strategy | N |

**总计**: 3 + N × 4 = 19-23次 Skill 调用

### 2.4 关键特性

- ✅ **断点续传**: 支持从中断处继续
- ✅ **自动重试**: 每个文档失败自动重试2-3次
- ✅ **错误隔离**: 单个文档失败不影响其他文档
- ✅ **进度保存**: 实时保存到 `.progress.json`

## 5. 重要提示：触发 batch-patent-generator 的关键

### ⚠️ 必须包含"批量"关键词

根据系统的意图检测规则，要触发 `batch-patent-generator` agent，**用户提示词必须明确包含"批量"关键词**。

### 正确的触发示例

| 示例 | 说明 |
|-----|------|
| "帮我批量生成智能垃圾分类系统的专利文档" | ✅ 包含"批量" |
| "批量生成专利和配套方案：智能垃圾分类系统" | ✅ 包含"批量" |
| "使用batch-patent-generator批量生成专利文档" | ✅ 包含"批量"和agent名称 |
| "我想批量创建一个完整的专利文档体系" | ✅ 包含"批量" |
| "批量生成技术交底书、专利权利要求书等完整专利文档" | ✅ 包含"批量" |

### ❌ 错误的触发示例（不会触发 batch-patent-generator）

| 示例 | 问题 |
|-----|------|
| "帮我生成专利文档" | ❌ 缺少"批量" |
| "生成智能垃圾分类系统的专利" | ❌ 缺少"批量" |
| "使用batch-patent-generator生成专利" | ❌ 缺少"批量" |
| "创建完整的专利文档体系" | ❌ 缺少"批量" |
| "我想生成一些专利相关的文档" | ❌ 缺少"批量" |

### 为什么需要"批量"关键词？

系统使用意图检测来决定调用哪个 agent：

1. **batch-patent-generator**: 需要包含"批量"关键词，用于**自动化批量生成**完整专利文档体系
2. **generate-docs-agent**: 不包含"批量"但包含文档生成意图，用于**单文档或依赖链生成**
3. **patent-challenger-agent**: 用于文档审核

### 测试中的使用

在本测试方案的所有触发示例中，都已确保包含"批量"关键词。请确保在实际测试中也遵循这一规则。

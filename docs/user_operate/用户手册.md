# 第一部分：快速入门

## 1. 系统简介

### 1.1 什么是 Claude Code UI

Claude Code UI 是一个为专利从业人员设计的 **AI 辅助文档生成平台**。它基于 Claude AI 模型，通过强大的 **Extensions 扩展系统**，帮助您：

- **快速生成专利文档**：技术交底书、权利要求书、商业价值分析等
- **撰写技术方案**：系统架构、实施方案、可行性评估
- **产出学术论文**：工程论文、科学论文、经济论文
- **处理文档格式**：PDF、Word、PowerPoint 文档的智能处理

### 1.2 核心功能概览

| 功能模块 | 说明 |
|---------|------|
| **智能对话** | 与 Claude AI 进行自然语言交互 |
| **Extensions 系统** | 预置的专业文档生成扩展 |
| **项目管理** | 多会话管理 |
| **文件浏览器** | 可视化项目文件管理 |
| **内置终端** | 容器隔离的安全执行环境 |

## 3. 5分钟快速上手

### 3.1 生成第一份技术交底书

在聊天输入框中输入：

```
请帮我生成一份技术交底书，主题是"基于物联网的智能绿植监测系统"
```

AI 会自动：
1. 识别您的意图为"专利文档生成"
2. 调用相应的扩展技能
3. 生成结构完整的技术交底书
4. 保存到右上角 files 标签页中的 `generated_docs/` 目录

### 3.2 查看生成的文件

1. 点击右侧的 **files** 标签
2. 展开 `generated_docs/` 目录
3. 找到生成的 Markdown 文件，点击查看

## 4. 项目与会话管理

> 当前系统使用默认工作区 `my-workspace`，无需切换项目

### 4.1 会话管理

#### 会话类型

系统支持三种 AI 提供商的会话：

- **Claude**：Anthropic Claude 模型（当前支持）
- **Cursor**：Cursor Codex 模型
- **OpenAI**：GPT 系列模型

#### 会话操作

| 操作 | 方法 |
|------|------|
| **新建会话** | 点击 **+ 新会话** |
| **切换会话** | 点击会话列表中的会话项 |
| **重命名会话** | 鼠标悬停 → 点击编辑图标 |
| **删除会话** | 鼠标悬停 → 点击删除图标 |

#### 会话历史

所有会话历史自动保存，可随时切换回历史会话继续对话。

## 5. 聊天与AI交互

### 5.1 发送消息

#### 基本操作

1. 在底部输入框输入消息
2. 按 **Enter** 发送
3. AI 响应会实时流式显示

#### 多行输入

按 **Shift + Enter** 换行，不发送消息。

#### Ctrl+Enter 发送模式

可在设置中开启，需要按 Ctrl+Enter 才发送。

### 5.2 文件引用

在对话中引用项目文件：

```
请分析 @generated_docs/requirement.md 中的内容
```

系统会：
1. 自动提示匹配的文件
2. 选中文件后自动插入引用
3. AI 获取文件内容进行分析

### 5.3 文件附件

支持上传文件并发送给 AI：

1. 点击输入框旁的 📎 图标
2. 选择本地文件
3. 文件会显示在预览区
4. 发送消息时一并提交

### 5.4 模型选择

点击输入框上方的模型选择器，可切换（当前仅支持glm-4.7）：

- **Claude Sonnet**：平衡性能和速度
- **Claude Opus**：最强推理能力
- **Claude Haiku**：快速响应

### 5.5 权限模式

点击权限模式按钮可切换：

| 模式 | 说明 |
|------|------|
| **默认** | AI 操作需要手动确认 |
| **接受编辑** | 自动接受文件编辑操作 |
| **绕过权限** | 不执行权限检查（谨慎使用） |
| **计划模式** | 仅规划操作，不执行 |

## 6. 文件浏览器使用

### 6.1 视图模式

文件浏览器支持三种视图：

| 模式 | 特点 |
|------|------|
| **简单视图** | 仅显示文件名，简洁 |
| **详细视图** | 显示大小、修改时间、权限 |
| **紧凑视图** | 单行显示关键信息 |

切换方式：点击视图切换按钮。

### 6.2 文件操作

#### 打开文件

- 单击文件名在编辑器中打开
- 代码文件支持语法高亮

#### 重命名文件

1. 悬停文件 → 点击 🗑️ 图标

直接拖拽文件到目标文件夹。

### 6.3 目录操作

#### 展开/折叠目录

# 第三部分：Extensions 深度指南

## 8. Extensions 系统概述

### 8.1 什么是 Extensions

Extensions（扩展）是系统的核心能力模块，通过以下五类组件提供专业功能：

| 扩展类型 | 图标 | 说明 | 示例 |
|---------|------|------|------|
| **Agents** | 🤖 | 智能编排器，协调其他组件 | generate-docs-agent |
| **Commands** | ⚡ | 快捷命令，以 `/` 开头 | /generate-docs |
| **Skills** | 🎯 | 具体技能，执行专业任务 | patent-writing |
| **Hooks** | 🪝 | 生命周期钩子，自动注入上下文 | subagent-context-injector |
| **Knowledge** | 📚 | 知识库，提供专业指导 | patent-guidelines |

### 8.2 Extensions 工作原理

```
您的输入
    ↓
意图识别 → 选择合适的 Agent/Skill
    ↓
上下文注入 → Hooks 自动加载相关知识
    ↓
任务执行 → 调用 Skill 生成文档
    ↓
质量评审 → 自动循环改进（可选）
    ↓
输出结果 → 保存到项目目录
```

### 8.3 如何调用 Extensions

有三种方式调用扩展功能：

| 方式 | 适用场景 | 示例 |
|------|---------|------|
| **自然语言描述** | 最常用，AI 自动识别 | "生成技术方案" |
| **使用命令** | 明确指定功能 | `#generate-docs 生成交底书` |

## 9. 文档生成功能

### 9.1 一句话说明

**所有 Skill 用法相同**：在聊天框中直接描述您的需求，AI 会自动识别并调用相应的扩展。

**生成任何一种文档示例**：

```
帮我完善下交底书：基于物联网的智能绿植监测系统

帮我生成交底书：基于物联网的智能绿植监测系统

请你根据创意生成技术方案；基于技术方案生成工程论文、科学论文、经济论文；从每篇论文挖掘1个新创意；为每个新创意生成交底书和商业价值报告
```

支持的文档类型如下（会随时更新，用户可以询问AI，最新支持生成的文档类型）：

### 9.2 专利类文档

| 文档类型 | 输入示例 |
|---------|----------|
| **技术交底书** | `请生成技术交底书：[创意描述]` |
| **权利要求书** | `请生成权利要求书：[技术方案]` |
| **专利创新评估** | `请评估专利创新性：[技术方案]` |
| **商业价值分析** | `请做专利商业价值分析：[创意或方案]` |
| **IP保护策略** | `请生成IP保护策略：[项目描述]` |

**批量生成专利文档链**：

```
请生成完整的专利申请文档链：[你的创意]

包括：技术方案、技术交底书、权利要求书、创新评估、商业分析、IP策略
```

系统会自动按依赖关系编排：

```
技术方案 → 技术交底书 → [权利要求书 + 创新评估 + 商业分析] → IP策略
```

### 9.3 技术类文档

| 文档类型 | 输入示例 |
|---------|----------|

| **可行性评估** | `请评估项目可行性：[项目信息]` |

| **合规标准策略** | `请生成合规标准策略：[项目描述]` |

### 9.4 学术论文

## 11. 工作流编排

### 11.1 串行执行

适用于文档间有先后依赖的场景：

```
先生成技术方案，然后基于技术方案生成交底书，最后生成权利要求书
```

执行流程：
```
技术方案 → 技术交底书 → 权利要求书
```

### 11.2 并行执行

适用于无依赖的独立文档：

```
同时生成：技术方案、商业价值分析、伦理风险评估
```

系统会同时发起多个任务，提高效率。

#### 使用 workflow 命令批量生成

使用 `/workflow` 命令可以更高效地批量生成文档。

**为多个创意分别生成文档**：

```
/workflow

创意1：智能水杯
创意2：无线充电器
创意3：智能笔筒
```

系统会为每个创意独立生成指定类型的文档。

**为同一创意生成多种文档**：

```
/workflow

创意：智能绿植监测系统
文档类型：技术交底书、商业价值分析、IP保护策略
```

系统会并行生成所有指定类型的文档。

> **提示**：使用 `/workflow` 命令时，系统会自动忽略对话中的其他内容，专注于批量生成任务。完成后可继续正常对话。

### 11.3 迭代执行

适用于从一组文档中提取创意并生成新文档：

```
从以下论文中各提取1个新创意，为每个创意生成交底书和商业分析

论文1：@paper1.pdf
论文2：@paper2.pdf
论文3：@paper3.pdf
```

### 11.4 复杂工作流

```
请执行以下工作流：

1. 根据创意生成技术方案
2. 基于技术方案生成工程论文、科学论文、经济论文
3. 从每篇论文挖掘1个新创意
4. 为每个新创意生成交底书和商业价值报告
```

系统会自动规划执行路径。

## 常见问题 FAQ

### Q1: 生成的文档保存在哪里？

A: 默认保存在聊天界面右上角 files 中的目录 `generated_docs/` 文件夹中，文件命名格式为 `[类型]_[创意简述]_[时间戳].md`。

### Q2: 如何编辑生成的文档？

A: 在文件浏览器中找到文件，点击打开。修改后点击保存。

### Q3: AI 生成的专利文档可以直接申请吗？

A: 生成的文档是草稿，建议由专业专利代理人审核后再提交申请。

### Q4: 如何提高生成质量？

A: 建议：
1. 提供详细的技术背景
2. 明确技术问题的描述
3. 补充技术细节和数据
4. 多次迭代生成

### Q5: 支持哪些文件格式？

A: 支持常见文本格式：
- Markdown (.md)
- 文本文件 (.txt)
- PDF (.pdf)
- Word (.docx)
- PowerPoint (.pptx)
- 代码文件（多种语言）

### Q6: 会话历史会丢失吗？

A: 不会。所有会话自动保存在数据库中，随时可以切换回历史会话。

### Q7: 如何同步扩展到所有用户？

A: 管理员可以在 **管理页面 → 扩展预置管理** 中执行同步操作。

### Q8: 批量生成时系统如何处理？

A: 系统会自动识别任务依赖关系：
- 有依赖的串行执行
- 无依赖的并行执行
- 批量的迭代执行
